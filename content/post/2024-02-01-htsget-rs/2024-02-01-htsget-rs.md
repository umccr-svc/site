# Outline

* Solve James Bonfield question/issues from GA4GH Slack thread: https://ga4gh.slack.com/archives/C024Q3W6YH3/p1701184709014209
Namely, James said:

```
(..) In short, it wasn't the format that mattered but the interface to that format.  In this case via tools.

We probably need to boost our support for http://samtools.github.io/hts-specs/htsget.html with VCF.  It appears to already support it in theory, 
with both region and sample subsetting, but in practice I'm not aware of servers that handle it.  I'm also not aware of tool chains that 
can directly query htsget via a URI just as if it was a local file.  For example (I haven't checked) I'd be very very surprised if 

bcftools view -s NA12878 htsget://some.server/joint.vcf 

actually fetched just NA12878 rather than fetching the entire VCF file and just filtered post-download.  

Fixing the tool chains and the servers may well be more important than codifying which specific VCF alternative we want to bless as a GA4GH standard long-term.
```

   * Agree with him that htsget could be the gateway to reduce our tight dependence with traditional and problematic formats such as VCF (interfaces vs formats).
   * Nit: unfortunately the decision to have `htsget://` as a protocol (as James uses in his example) is stuck in discussion in https://github.com/samtools/hts-specs/issues/581 since 2021 :/
   * Oneliner/curl demo against the UMCCR demo server.
   * Make sure that the VCF is not transferred in bulk as James points out, but only the regions queried. Offer profiling aids?

* Crypt4GH support in the works, describe collab with NBIS and CSC... [point to Draft PR][crypt4gh-draft-pr] and the current key exchange protocol demo.
* Ownership of crypt4gh-rust on crates.io... waiting for ownership and merge the pullrequest over EGA-Archive GitHub.

* Future:

1. wasm-bindgen for crypt4gh-rust crate and being able to use it on browser WASM library for viz frontends such as igv.js, JBrowse2, etc...
2. htsget-portal, lightweight JWT logic, potentially connected with GA4GH's passports efforts.
